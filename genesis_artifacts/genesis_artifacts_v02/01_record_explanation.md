**AnchorForge Integrity Record (Inception Payload)**

This JSON object is a complete, self-contained SPV proof generated by AnchorForge. It includes:
- The original anchored data (`original_audit_content` / `original_file_hash`)
- The raw blockchain transaction (`raw_tx`)
- The Transaction ID (`txid`)
- The Merkle path (`merkle_proof`), acquired after block confirmation.

Together with a local block header, this structure allows cryptographically secure, off-chain verification of the data's integrity and timestamp. 
This specific record serves as the payload for the AnchorForge Genesis 2 (Inception) transaction, demonstrating the protocol's ability to secure complex, nested audit trails.

**AnchorForge v0.2: TLV Payload Decoder**

This Integrity Record contains a `raw_tx` hex string. To parse the AnchorForge OP_RETURN payload, look for `OP_FALSE OP_RETURN` followed by these Tag-Length-Value (TLV) blocks. 

**Tags (Hex / ASCII):**
* `f0` (AppID): Identifies the AnchorForge protocol and version.
* `48` ('H'): Hash of the anchored data (Prefix `00` = SHA256).
* `50` ('P'): Public Key for ECDSA (Prefix `00` = Compressed).
* `53` ('S'): ECDSA Signature (Prefix `01` = DER format).
* `43` ('C'): X.509 Certificate (Prefix `01` = PEM format).
* `44` ('D'): Embedded On-Chain Data (Prefix `00` = UTF-8, `01` = RAW).
* `4e` ('N'): Transaction Note (Plain text).
* `58` ('X'): External Reference / URI.

**Parsing Rule:** [Push-Opcode] + [TAG] + [Length-Opcode] + [Prefix-Byte] + [Value]
